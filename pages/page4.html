CSS

CSS allows you to change how your website looks, so let's go over it!

Initialise CSS

I said, "Every" tag has to be inside the <body>-Tag, well for CSS and Javascript this doesnt apply, so let's break some rules here!

To start using CSS we build a <style>-Tag outside of the <body>-Tag.
Kinda like:

So, now that we have a CSS-Sheet in our code, we should go over the basics of CSS.

In CSS we can define spefic ways an Element can look with either the tags, Id's of the element or so called Classes.
Both class and Id are an attribute any HTML-Tag can have, (<div id="someID" class="TestClass Testclass2">).
As you can see, you can also combine multiple classes in a single Element, so you can be very specific with classes.

lets start creating a Class, in your Stylesheet (<style>) you can create a new class by writing:
.Classname
{

}

The Dot here specifies that we are talking about a class.
To use this class give any HTML-Tag the attribute class="Classname". To use multiple classes just write two classnames in the same attribute with a space between them.

To Alter a whole tag, (so everytime for example, when you use <img>, the image looks exactly how you want it to look), you just get rid of the Dot.
Kinda like:
img
{

}

And to use an ID you just use a #
#IDname
{

}

if you want to just target the whole page, you write a * instead of an ID, Class or Tag (nothing else needed)

Options

So lets start altering stuff with CSS now, the syntax of a Stylesheet is simple
.SomeClass
{
SomeKindofAttribute : Value;
}
the Semicolon (;) after every property is mandatory.

Here are all properties a Tag can have, but I'll explain the most important ones and give some extra trivia for some.
color

Very Simple, you can change the Color of the written Text, you can use predefined colors, rgb codes like "rgb(0, 0, 0)" (which would be black) or hex like "#000000" (also black).
rgb also has a sub category called rgba in which you can add an opacity aswell like "rgba(0,0,0.5)"

color: tomato

background-color

You can set the background-color of your element to any color you want, "transparent" is the default value.

background-color: aquamarine
text-align

As the title says, you use this to align text either to the left, right or center etc.

text-align: right

text-align: center

text-align: left

background-image

basically the same as the background-color, just that you specify url(link) instead of an color.

background-image: url(some_image_link)
border

As the name says, you can put a border around anything. with border you can set the width, color and style without any other properties, but also WITH other properties.
You can also write them in whatever Order you want

border: 2px tomato solid

border: 2px tomato dashed

border: 4px tomato double

border: 4px tomato ridge

font

there are multiple properties to manipulate a font. there is font, in which you can manipulate basically anything.

font: 15px Arial, sans-serif;
This is a paragraph. The font size is set to 15 pixels, and the font family is Arial.

font: italic small-caps bold 12px/30px Georgia, serif;
This is a paragraph. The font is set to italic and bold, with small-caps (all lowercase letters are converted to uppercase). The font size is set to 12 pixels, the line height is set to 30 pixels, and the font family is Georgia.

Or you can just manipulate single font attributes with font-size, font-weight, font-family and font-style (and more.)

font-size: x-large;
font-family: 'Times New Roman', Times, serif;
font-weight: 700;
font-style: italic;

more about fonts
height & width

You can set the height and width of any Element, with these simple properties, you just need to give a measurement, use px (pixel) to hardcode them, % (in relationship to the parent-element aka container above it) or vw/vh for viewwidth/viewheight which is the relationship to the browser fenster, also works like % so 100vh/vw should cover the whole screen

width: 75vw; height: 25vh

margin & padding

I decided to put these two together since you will most of the time use one or both of them for easy positioning (we'll talk about absolute positioning later.)
Both can also just count for a single direction the syntax would be margin-direction or padding-direction
Example: margin-bottom | padding-left

Margin is a spacer between two elements, while padding just extends the element a bit more.

the green lines are the padding here, it also has a margin-left (away from left) of 5px

display

Display configures how the Element behaves inside it's containers, and how it aligns itself next to the other elements in the same container.
You can also just let a whole Element dissapear with display: none;
More Information about Display

Overflow

with overflow you can manipulate how elements look, once they would go over the parent container, for example you can hide anything past the regular screen size with
body
{
overflow: none;
}
There are also overflow-y and overflow-z variants for only specific axis's.

Pseudo Classes.

You can also manipulate more specific classes in your stylesheet, with multiple methods.
If you write a new class like ".class_1.class_2" in your stylesheet, you will manipulate anything that has both class_1 and class_2 at the same time.
.class1.class2
{

}

Important to know is, that all containers transfer the classes to their children elements, so if you have a
<div class="class1"> <p class="class2"> </p> </div>, the P-Element will still take the attributes of Class1.

To specify a path of classes, (let's use the same example from above), to specify class2 INSIDE of class 1 you would write it with a space between the two classes

.class1 .class2
{

}

Important to Note is also that this can be extended, and class1 wouldnt have to be the direct parent of class2 here, it just needs to be above it. (The Last class is always the one which gets changed)

If two classes will share the exact same properties you can combine them with a comma like

.class1,
.class2
{

}

Now both of these classes will have the properties set within.

While these are still kind of pseudo Class declarations, actual Pseudo Classes are rather about the "state" the class is in.
you declare them with a :
kind of like:

.class1:hover
{

}

Now everything inside here will only apply once you hover any Element with Class1. all other style-properties will be taken from the normal .class1 style.
It will use the standard style for the element if no "normal" tag is set, so you can use Pseudo classes without the normal ones existing too!

Here are all Pseudo-Classes in CSS. (I don't know them all either lmfao)

The only ones which you will definetly need are :hover (self explaining), :active (if you press a button for example, the button will be active while pressed.) and :has(.class1)

"has" is an interesting one, since you can basically ask if your specific tag has another specific parent tag at some point but not if you have a specific child Element.
so You can use :has(childCLASS) to manipulate the parent, if it has that specific class inside of it.
<div class="class1"> <div class="class2"> <p class="class3"> </p> </div></div>
So to ask if Class2 inside of Class1 contains a Class3 class you would do something like this.

.class1 .class2:has(.class3)
{

}

More to :has()

So as you can see, you can get VERY specific with these Classes
